# 发布说明

## 用途

此目录用于存放导出后的 txt、整卷版本和平台发布版。

## 原则

- 正文源文件以 `4_正文源码/` 为准。
- 发布版本只作为导出结果，不反向修改正文源文件。
- 平台发布前应检查章节标题、空行、字数和敏感表达。
- 每个 `.txt` 发布文件旁边必须生成一个同内容 Markdown 预览文件，文件名在原名后追加 `_md.md`。
- `9_发布版本/平台发布版/` 必须按卷分目录，不允许所有平台章节平铺在根目录。
- 平台发布版文件名必须同时标明卷名与章节号，章节号按卷内三位数字编号。
- 每一卷最多 999 章；第二卷、第三卷等新卷重新从 `第001章` 开始。

示例：

```plaintext
9_发布版本/第一卷/第001章_十二小时预警.txt
9_发布版本/第一卷/第001章_十二小时预警_md.md

9_发布版本/平台发布版/第一卷_灾变降临/第一卷_第001章_十二小时预警.txt
9_发布版本/平台发布版/第一卷_灾变降临/第一卷_第001章_十二小时预警_md.md
```

`.txt` 用于平台上传。

`*_md.md` 用于 Obsidian 中浏览发布版内容。

审稿记录、批注处理记录、开头方案、废稿不进入发布版本。

字数统计只保存在 `7_长期记忆/字数统计.md`，不得进入发布版本。

## 当前转换规则

正文源文件允许包含 Obsidian 与 AI 工程上下文标识。

导出发布版本时必须去除：

- YAML frontmatter。
- HTML 注释批注，例如 `<!-- 批注: ... -->` 或 `<!-- 批注开始: ... -->...<!-- 批注结束 -->`。
- `## 关联索引`。
- `## 本章推进`。
- `## 新伏笔`。
- `## 情绪节奏`。
- `## 字数统计`。
- Obsidian 双链标记，`[[李阳]]` 转为 `李阳`，`[[路径|显示名]]` 转为 `显示名`。
- Markdown 标题符号，`# 第001章 标题` 转为 `第001章 标题`。

当前转换脚本：

```plaintext
tools/clean_markdown.py
```

当前整卷构建脚本：

```plaintext
tools/build_volume.py
```

当前一键导出脚本：

```plaintext
tools/export.py
```

## 平台发布版命名规则

平台发布版固定使用如下结构：

```plaintext
9_发布版本/平台发布版/
├── 第一卷_灾变降临/
│   ├── 第一卷_第001章_十二小时预警.txt
│   └── 第一卷_第001章_十二小时预警_md.md
└── 第二卷/
```

规则：

- 卷目录使用 `第X卷_卷名`；如果卷名暂未确定，先使用 `第X卷`。
- 平台发布文件使用 `第X卷_第001章_章节名.txt`。
- 章节号按卷内三位数字编号：每卷从 `第001章` 开始，到 `第999章` 为止。
- 如果单卷规划超过 999 章，必须拆分新卷；新卷重新从 `第001章` 开始。
- 平台发布版的 `_md.md` 文件必须与同名 `.txt` 内容完全一致，仅用于 Obsidian 浏览。
- 原始卷目录如 `9_发布版本/第一卷/` 可保留源码章节命名；平台发布版必须使用上述长期稳定命名。

## 规则同步要求

如果后续在 `4_正文源码/` 的章节中新增新的前缀、后缀、块标记或工程上下文段落，必须同步更新：

- `tools/clean_markdown.py`
- 本文件的转换规则说明
