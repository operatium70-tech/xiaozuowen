# AI协作说明

## 协作目标

AI 的任务不是替代作者，而是帮助维护一个长期稳定的小说工程。

AI 应协助完成：

- 世界观一致性检查
- 章节大纲拆解
- 伏笔投放与回收
- 人物动机校验
- 情绪节奏控制
- 正文草稿生成
- 平台发布版整理

## 推荐读取顺序

写作前优先读取：

1. `0_系统规则/`
2. `7_长期记忆/`
3. `1_世界观/`
4. `2_人物设定/`
5. `3_剧情大纲/`
6. `5_文风控制/`
7. `8_技能规则/`

## 输出要求

AI 输出正文时，应保持：

- 冷峻
- 克制
- 压迫感
- 信息差
- 生存感
- 长线悬疑

AI 输出设定时，应标明：

- 已确认
- 待确认
- 禁止直接写入正文
- 可作为伏笔

## 上下文维护

每次完成章节、卷纲或重大设定后，应同步更新 `7_长期记忆/`。

## 规则同步要求

当作者要求新增、修改或确认任何规则时，AI 不能只修改单个文件，必须先判断影响范围。

常见影响范围：

- 项目总入口：`README.md`
- 系统规则：`0_系统规则/`
- 世界观与设定：`1_世界观/`
- 剧情规划与伏笔：`3_剧情大纲/`
- 正文格式规范：`4_正文源码/正文工程规范.md`
- 文风与章节规格：`5_文风控制/`
- 长期记忆：`7_长期记忆/`
- 技能规则：`8_技能规则/`
- 发布说明：`9_发布版本/发布说明.md`
- 导出脚本：`tools/clean_markdown.py`、`tools/export.py`、`tools/build_volume.py`、`tools/word_count.py`

如果新规则影响发布版格式、文件命名、目录结构或正文清理逻辑，必须同步更新发布说明和对应脚本，并重新执行：

```bash
python3 tools/export.py
```

导出后必须检查：

- 发布版没有批注。
- 发布版没有 YAML frontmatter。
- 发布版没有 Obsidian 双链标记。
- 发布版没有 AI 工程上下文段落。
- `.txt` 与同名 `_md.md` 内容一致。

## 字数统计要求

章节新增、删除、定稿或大幅修改后，AI 必须同步更新字数统计。

执行命令：

```bash
python3 tools/word_count.py
```

统计结果固定写入：

```plaintext
7_长期记忆/字数统计.md
```

字数统计必须包含：

- 每章字数。
- 每卷字数。
- 全书当前总字数。

字数统计只属于工程管理和长期记忆，不得写入正式章节正文，不得进入 `9_发布版本/`。

## 状态同步要求

章节新增、定稿、重命名、删除或完成大改后，AI 必须同步检查：

- 对应卷的 `卷说明.md`。
- `7_长期记忆/当前剧情状态.md`。
- `7_长期记忆/最近20章总结.md`。
- `7_长期记忆/当前时间线.md`。
- `7_长期记忆/字数统计.md`。
- `3_剧情大纲/` 中相关卷纲、主线、伏笔记录。
- `9_发布版本/` 中对应发布文件。
