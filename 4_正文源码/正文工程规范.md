# 正文工程规范

## Obsidian 图谱标识

正式章节必须使用两类 Obsidian 标识：

1. YAML frontmatter：用于按卷、章节、状态、标签筛选。
2. 正文内 `[[双链]]`：用于 Obsidian 关系图谱。

推荐结构：

```markdown
---
类型: 正文章节
卷: 第一卷_灾变降临
章节: 第001章
标题: 十二小时预警
状态: 草稿
tags:
  - 正文
  - 第一卷
---

# 第001章 十二小时预警

## 关联索引

- 主角：[[李阳]]
- AI：[[AI来源]]
- 灾变：[[灾变设定]]
```

## 临时稿处理

开头方案、废稿、试写片段等临时文件，在正式版本确定后应删除。

如果需要保留废稿，应移入：

```plaintext
4_正文源码/废弃章节/
```

## 发布版同步

`4_正文源码/` 中正式章节更新后，必须同步更新 `9_发布版本/`。

当前同步目标：

- `9_发布版本/第一卷/`
- `9_发布版本/平台发布版/`
- `9_发布版本/全卷整合版/`

当前导出命令：

```bash
python3 tools/export.py
```

## 转换规则同步

正文源文件允许保留 AI 上下文、Obsidian 标识和工程备注。

但任何新增标记都必须同步更新：

- `tools/clean_markdown.py`
- `9_发布版本/发布说明.md`

否则不得进入正式章节。
