# 正文工程规范

## Obsidian 图谱标识

正式章节必须使用两类 Obsidian 标识：

1. YAML frontmatter：用于按卷、章节、状态、标签筛选。
2. 正文内 `[[双链]]`：用于 Obsidian 关系图谱。

推荐结构：

```markdown
---
类型: 正文章节
卷: 第一卷_灾变降临
章节: 第001章
标题: 十二小时预警
状态: 草稿
tags:
  - 正文
  - 第一卷
---

# 第001章 十二小时预警

## 关联索引

- 主角：[[李阳]]
- 配角：[[队伍同行/林小满|林小满]]、[[队伍同行/赵建国|赵建国]]
- AI：[[AI来源]]
- 灾变：[[灾变设定]]
- 伏笔：[[伏笔系统]]
- 大纲：[[主线剧情]]
- 时间线：[[当前时间线]]
- 字数：2029
```

### 关联索引规则

每章 `## 关联索引` 必须包含：

- `主角`：本章主视角或核心主角，当前默认 `[[李阳]]`。
- `配角`：记录本章实际出场，或虽未出场但对本章推进有直接影响的重要提及人物。
- `AI`、`灾变`、`伏笔`、`大纲`、`时间线`：用于维护 Obsidian 图谱。
- `字数`：使用 `python3 tools/word_count.py` 清理后的章节正文字数。

配角索引规则：

- 可复用人物优先使用 Obsidian 双链，例如 `[[队伍同行/林小满|林小满]]`、`[[远程线索/周凯|周凯]]`。
- 配角档案按叙事功能归入 `2_人物设定/配角/` 下的分类目录；章节索引引用时优先使用带路径和别名的双链。
- 一次性功能路人可用普通文本，例如 `遇袭邻居`、`断腿女人`。
- 没有可复用配角时写 `配角：无`。
- 未命名但后续可能复用的人物，可先用功能名建档，例如 `[[幸存者群像/父子幸存者|父子幸存者]]`；正文正式命名后再同步重命名。

章节正文大幅修改后，必须同步更新该章 `字数` 行、运行 `python3 tools/word_count.py`，并重新导出发布版。

## 临时稿处理

开头方案、废稿、试写片段等临时文件，在正式版本确定后应删除。

如果需要保留废稿，应移入：

```plaintext
4_正文源码/废弃章节/
```

## 审稿批注

作者在 Obsidian 中审稿时，推荐使用 HTML 注释批注。

短批注：

```markdown
<!-- 批注: 这里节奏偏快，建议补一笔心理反应 -->
```

圈定段落：

```markdown
<!-- 批注开始: 这一段逻辑不顺，需要调整 -->
原正文段落
<!-- 批注结束 -->
```

批注处理后应删除注释。

导出发布版时，`tools/clean_markdown.py` 会自动剥离 HTML 注释，避免批注进入 `9_发布版本/`。

### 批注处理规则

所有批注必须逐条回应。

处理批注时不能默认“批注即必须修改”。每条批注需要判断：

- 是否采纳。
- 是否修改正文。
- 修改位置。
- 不修改时的合理解释。

整体批注优先级高于局部句子批注；如果整体时间线、设定逻辑或人物动机存在问题，应先处理整体问题，再处理局部表达。

### 批注记录存放规则

每个卷目录下应单独建立 `批注记录/` 子目录，用于存放该卷所有批注处理记录。

批注处理记录不得与正式章节文件混放在卷目录根部。新增记录统一命名为：

```plaintext
批注记录/第001章_批注处理记录.md
批注记录/第002章_批注处理记录.md
```

卷说明、长期记忆或其他索引引用批注处理记录时，应指向 `批注记录/` 子目录，例如：

```markdown
[[批注记录/第001章_批注处理记录]]
```

`批注记录/` 属于审稿工程资料，不进入发布版，不参与正文导出和全卷整合。

## 发布版同步

`4_正文源码/` 中正式章节更新后，必须同步更新 `9_发布版本/`。

当前同步目标：

- `9_发布版本/第一卷/`
- `9_发布版本/平台发布版/第一卷_灾变降临/`
- `9_发布版本/全卷整合版/`

平台发布版必须按卷分目录，并使用长期稳定文件名：

```plaintext
9_发布版本/平台发布版/第一卷_灾变降临/第一卷_第001章_十二小时预警.txt
9_发布版本/平台发布版/第一卷_灾变降临/第一卷_第001章_十二小时预警_md.md
```

章节号按卷内三位数字编号；每卷从 `第001章` 开始，到 `第999章` 为止。第二卷、第三卷重新从 `第001章` 开始；如果单卷超过 999 章，应拆分新卷，不得改成四位数或跨卷累计编号。

当前导出命令：

```bash
python3 tools/export.py
```

## 字数统计同步

章节新增、删除、定稿或大幅修改后，必须同步更新字数统计：

```bash
python3 tools/word_count.py
```

统计结果固定写入：

```plaintext
7_长期记忆/字数统计.md
```

字数统计必须能看到：

- 每章已写字数。
- 每卷已写字数。
- 全书当前总字数。

字数统计属于工程管理层，不能写入正式章节正文，不能进入发布版。

## 卷说明同步

每次正式章节新增、删除、重命名或完成重要修改后，必须同步更新对应卷目录下的 `卷说明.md`。

至少更新：

- 当前状态。
- 正式章节表。
- 本卷当前进度。
- 下一步。
- 发布版同步状态。

## 转换规则同步

正文源文件允许保留 AI 上下文、Obsidian 标识和工程备注。

但任何新增标记都必须同步更新：

- `tools/clean_markdown.py`
- `9_发布版本/发布说明.md`

否则不得进入正式章节。
