# 章节生产模板

## 实现原理

本模板规定未来每章正文开写前后必须填写的结构项，防止 AI 脱离设定自由发挥。

## 如何使用

用户明确要求写正文后，先用本模板做章节准备，再写正文。当前阶段仍不写小说正文。

本文已由用户于 2026-05-24 同意确认，可作为后续写章前模板；当前仍需用户明确要求写正文后才执行。

## 写章前模板

| 项目 | 必填内容 |
| --- | --- |
| 本章编号 | 使用中文章名或章节编号，不在文件名前加数字前缀 |
| 所属阶段 | 读取对应阶段总纲、功能定义、推进表 |
| 本章目标 | 本章只解决一个主问题 |
| 主冲突 | 人物、规则、资源、情绪或组织压力 |
| 参与人物 | 每个人必须有功能，不做背景板 |
| 本章伏笔 | 最多一个主伏笔，必须先服务当前剧情 |
| 本章代价 | 付出时间、资源、关系、名声、伤势或责任 |
| 禁止跑偏 | 从对应设定文件复制本章禁区 |
| 章末钩子 | 指向下一章压力，不靠强行悬念 |

## 写章后同步

每章完成后必须同步：

1. [[长期记忆/当前状态/当前剧情状态|当前剧情状态]]
2. [[长期记忆/当前状态/最近章节总结|最近章节总结]]
3. [[长期记忆/当前状态/当前时间线|当前时间线]]
4. [[长期记忆/当前状态/当前伏笔状态|当前伏笔状态]]
5. [[长期记忆/当前状态/当前战力状态|当前战力状态]]
6. [[长期记忆/当前状态/当前人物关系|当前人物关系]]
7. [[长期记忆/当前状态/当前世界变化|当前世界变化]]
8. [[长期记忆/当前状态/字数统计|字数统计]]

## 章节质量门槛

1. 一章只能有一个主目标。
2. 每章必须有外部压力和内部变化。
3. 每章必须让至少一个长期线推进一点。
4. 任何爽点必须有代价或后续反噬。
5. 人物不能只为解释设定而出现。

## 禁止跑偏

1. 未经用户明确要求，不写正文。
2. 未完成写后同步，不宣布章节完成。
3. 不把未来新增未确认内容写成已确认正史。
4. 不用系统、老爷爷、神体、天降机缘解决章节问题。
