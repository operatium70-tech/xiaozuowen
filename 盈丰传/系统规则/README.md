# 系统规则

## 实现原理

`系统规则/` 存放长期稳定的工程约束，用来规定 AI 如何协作、如何判定正史、如何处理冲突、如何同步文件。

## 如何使用

每次任务先读取本目录，再读取世界观、人物、剧情和长期记忆。

## 文件职责

| 文件 | 职责 |
| --- | --- |
| `AI协作说明.md` | AI 协作边界、读取顺序、通知要求 |
| `最终设定硬规则.md` | 当前已锁定的硬设定和硬边界 |
| `当前硬边界.md` | 不能触碰的红线 |
| `正史优先级与冲突处理.md` | 资料冲突时的判定顺序 |
| `章节流程与同步规则.md` | 未来章节写作后的同步闭环 |

## 权威顺序

1. 用户本轮明确指令。
2. `global.txt` 与本目录 P0 规则。
3. 已确认新版正史。
4. 已写入正文且确定保留的事实。
5. 世界观、人物设定、剧情定稿。
6. 长期记忆当前状态。
7. 参考资料和临时建议。
