# 第三阶段剧情推进总表

## 实现原理

本表把第三阶段拆成可继续细化的推进段落，只记录功能和因果，不写小说正文。

## 如何使用

后续拆第三阶段卷纲或十章细纲时，以本表确认每一段的任务、赵盈丰变化、外部压力和阶段出口。

## 推进总表

| 阶段段落 | 建议范围 | 主线任务 | 赵盈丰变化 | 外部压力 |
| --- | --- | --- | --- | --- |
| 母星级入口 | 一百三十六至一百四十五章 | 以低位观察资格进入母星级项目 | 从被持续观察转向证明稳定性 | 项目审查、军方记录、学院背书 |
| 复核标准试点 | 一百四十六至一百六十章 | 建立元素异常复核流程和旧装备维护标准 | 从个人判断转向流程化 | 样本复杂、跨机构不信任 |
| 家族重评 | 一百六十一至一百七十五章 | 赵氏家族重新评估旁支价值 | 从被轻视转向被计算 | 嫡系秩序、授权风险、家族会议 |
| 跨大陆任务 | 一百七十六至一百九十五章 | 在不同区域元素条件下修正模型 | 从熟悉环境有效转向异地修正 | 区域差异、地方机构、商会利益 |
| 项目失效 | 一百九十六至二百一十章 | 原有流程在复杂环境下失效，出现损失 | 从建立流程转向承担后果 | 小队伤损、责任追溯、舆论压力 |
| 局部规则成型 | 二百一十一至二百二十五章 | 修正流程，建立可交接的小系统 | 从个人方法转向局部规则 | 审计、家族授权、长期维护成本 |
| 阶段出口 | 二百二十六至二百三十五章 | 获得小规模独立授权或跨大陆技术试点 | 得到下一层最低入场券 | 仍非家族核心，仍需外部验证 |

## 主线因果

第一阶段让赵盈丰被看见。

第二阶段让赵盈丰学会让组织相信他。

第三阶段让赵盈丰把“组织愿意相信他”升级成“组织可以依赖他设计的流程”。

如果第二阶段主题是“有用必须被组织接住”，第三阶段主题就是“能被接住，还要能长期运行”。

## 关键转折

### 项目入口不是奖励

赵盈丰进入母星级项目，不是被赏识后平步青云，而是被放到更严格的审查系统中。

他必须证明自己的方法不是偶然高光。

### 家族重评不是打脸

赵氏家族重新评估他时，重点不是道歉，而是算账：他能带来多少价值、承担多少风险、是否值得授权。

### 跨大陆任务让模型失效

跨大陆环境中，元素丰度、仪器标准、地方制度和商会习惯都不同。赵盈丰旧模型必须失效一次。

### 责任第一次落到他身上

第三阶段必须让赵盈丰承担一次真实责任：流程按他的思路执行，却仍然出错。

这不是为了压他，而是让他理解局部规则意味着长期后果。

## 与前两阶段的承接

| 前期伏线 | 第三阶段承接 |
| --- | --- |
| 仪器误差 | 元素异常复核流程 |
| 旧护具 | 低成本装备维护标准 |
| 证据链训练 | 跨机构审计报告 |
| 商会包装 | 技术归属和合同边界 |
| 军方评价 | 母星级项目的纪律要求 |
| 赵倩倩成熟分离 | 现实裂缝、家庭规划差异和体面放手 |
| 沈知微低谷入口 | 项目失效后的复盘听证、救援记录和心理风险记录 |

## 禁止跑偏

1. 不让筑基境能力细节提前定稿。
2. 不写成家族集体后悔的打脸剧情。
3. 不让赵盈丰靠战力解决跨大陆任务。
4. 不让项目失效只为虐主，必须推动方法升级。
5. 不让阶段出口变成家族核心身份。
