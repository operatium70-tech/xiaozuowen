# 阶段功能定义

## 实现原理

本文件把第三阶段的功能拆成修炼、社会、家族、项目和感情五条线，防止 [[剧情大纲/定稿/第三阶段筑基境/阶段总纲|第三阶段筑基境]] 被写成单纯升级。

## 如何使用

拆第三阶段事件时，先判断该事件推动哪一条功能线。若事件只提供战斗爽点，不改变赵盈丰的方法、身份、责任或关系，就不应保留。

## 修炼功能

第三阶段只确认功能，不定稿具体能力。

筑基境在剧情中的功能是：稳定个人逆熵基底。对应到叙事上，就是 [[人物设定/赵盈丰|赵盈丰]] 的方法不再依赖临场聪明，而要形成稳定、可复核、可交接的小系统。

本阶段应展示：

1. 他能更稳定地维持自身状态，但仍需要代价。
2. 他能让工具、流程和团队承接自己的判断。
3. 他开始从“发现异常”走向“建立异常处理标准”。
4. 他仍不能绕开资源、组织授权、家族信用和责任边界。

## 社会功能

第三阶段要让赵盈丰看见：社会承认不是一次资格，而是长期信用。

第二阶段的低位观察资格，只能证明他值得被看一眼；第三阶段必须证明他能被持续委托。

社会功能包括：

1. 母星级项目对他提出稳定性要求。
2. 跨大陆环境对他的模型提出修正要求。
3. 地方机构和商会要求他的方案可审计、可复用、可追责。
4. 军方或学院要求他的团队流程能经受失败。

## 家族功能

赵氏家族重评不能写成打脸。

重评的重点不是“他们终于知道赵盈丰厉害”，而是家族开始计算：这个旁支是否能形成可持续价值，是否值得给小规模授权，是否会给嫡系秩序带来不稳定。

家族内部应存在三种态度：

1. 功利支持：认为他能给家族带来项目价值。
2. 审慎观察：认可成果，但担心授权风险。
3. 秩序防御：担心旁支跃升破坏既有评价体系。

这些都不应写成恶毒反派。

## 项目功能

第三阶段最适合设计一个“低成本元素复核与装备维护试点”类项目。

项目功能是把前两阶段的伏笔串起来：

1. 仪器误差变成复核流程。
2. 旧护具改造变成维护标准。
3. 证据链报告变成审计记录。
4. 战场纪律变成小队撤退标准。
5. 商会包装变成技术归属与合同条款。

## 感情功能

第三阶段要完成赵倩倩线的成熟分离，但不能写成强行分手。

[[人物设定/赵倩倩|赵倩倩]] 的路线更稳定、更被家族支持，赵盈丰的路线更曲折、更依赖项目和外部授权。两人仍可互相尊重，但生活节奏、家庭参与程度、未来规划和社会压力逐渐不同。

本阶段应让两人共同承认现实结构不可长期共生，并保留互相尊重。分离完成后，沈知微可以通过项目失效后的复盘听证、救援记录和心理风险记录自然进入，不把她写成疗伤替身。

## 失败功能

第三阶段必须安排方法失效。

建议至少保留三类失败：

1. 标准失效：原有复核流程在跨大陆环境下出现偏差。
2. 授权失效：项目需要家族或机构授权，但审批比技术更难。
3. 责任失效：小队按他的流程执行后仍出现损失，他必须承担后果。

失败的作用是让赵盈丰理解：建立规则比借规则更难。

## 阶段终点功能

第三阶段终点应给赵盈丰一个小规模独立授权。

这个授权可以是跨大陆技术试点、小队长期任务、家族外部项目、学院与商会联合复核标准，但不能让他一步成为赵氏核心。
