# 开写前总审

## 实现原理

本文件用于在进入正文前做一次总审，只检查设定、人物、阶段、伏笔和执行层是否能闭环，不写小说正文。

## 如何使用

每次准备开写正文前，先读 [[总控制大屏]]、[[系统规则/开写白名单|开写白名单]]、[[系统规则/最终设定硬规则|最终设定硬规则]] 和本文件。

本文已由用户于 2026-05-24 同意确认，可作为后续拆纲依据，但不得直接写成小说正文。

## 冲突检查

| 方向 | 风险 | 修正原则 |
| --- | --- | --- |
| 修炼升级 | 后期容易战力化 | 每次升级必须先写认知精度、控制尺度和逆熵稳定度 |
| 感情线 | 容易写成后宫或前任贬低 | 四段感情各自完成一条相处原则，周燕线不写成奖励 |
| 家族线 | 容易写成打脸 | 赵氏重评是利益和责任核算，不是集体后悔 |
| 组织线 | 容易嘴炮胜利 | 标准和规则来自证据、责任、成本和可执行性 |
| 伏笔线 | 容易提前揭底 | 每个伏笔先服务当前剧情，再服务长线 |
| 地图线 | 容易随机换地图 | 每个新地图必须有元素生态、文明规则和资源差异 |

## 开写前最低门槛

1. 第一章必须能同时服务赵盈丰处境、元素误差、表达缺陷和比赛入口。
2. 前十章必须完成从“旁门杂学”到“被关注但有隐患”的起势。
3. 第一阶段必须让赵盈丰压线进入替补，而不是冠军登顶。
4. 每十章至少推进一条人物线、一条规则线和一个伏笔。
5. 每章禁止只写爽点，必须留下代价、误判或下一步压力。

## 禁止跑偏

1. 不把本文件写成正文。
2. 不把未来新增未确认内容直接写入已确认硬规则。
3. 不绕过 [[剧情大纲/待确认问题|待确认问题归档]]。
4. 不把高阶能力、地图和对手系统提前当作最终定稿。
