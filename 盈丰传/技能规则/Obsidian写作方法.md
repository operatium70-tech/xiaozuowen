# Obsidian写作方法

## 实现原理

Obsidian 方法就是用文件夹、双链、图谱，把小说设定、人物、剧情全部网状关联。AI 只能按规则写，彻底杜绝崩设定、忘伏笔、OOC。

## 如何使用

新增任何设定时，必须放入对应层级，并补充与人物、世界观、剧情、长期记忆的双链。

## 宏观设计理念

用 AI 写小说时：

- 作者 = 定规则、定世界观、定人设、定节奏。
- AI = 纯打字机、严格执行、零自创。
- 所有自由度全部锁死，只留执行权给 AI。

## 文件夹架构

| 目录 | 作用 |
|---|---|
| `系统规则` | 最高优先级，定义边界和约束 |
| `世界观` | 世界设定，所有内容的基础 |
| `人物设定` | 角色档案，行为逻辑来源 |
| `剧情大纲` | 路线图，剧情推进依据 |
| `正文源码` | 创作层，唯一输出源 |
| `文风控制` | 风格约束，保证一致性 |
| `参考资料` | 输入层，灵感来源 |
| `长期记忆` | 状态层，上下文保持 |
| `技能规则` | 能力层，AI 专项技能 |

## 必须维护的链接

1. 人物文件必须链接对应剧情线、境界和关系对象。
2. 世界观文件必须链接对应境界、修炼本质和资源规则。
3. 剧情文件必须链接相关人物、阶段和硬边界。
4. 长期记忆必须保留已确认记录和待确认问题。
