# 宏观设计理念

> 本项目的核心设计思想、架构哲学和工程化方法论。

---

## 项目定位

**"这不是存放小说的文件夹，而是一个长期可维护、可扩展、适合 AI 持续协作的小说工程。"**

核心理念：把小说创作从"灵感驱动"转变为"工程化管理"。

---

## 三大设计支柱

### 1. 分层架构

内容按职责分层存放，目录禁止数字编号前缀：

```
总控制大屏 → 项目总入口，所有协作先从这里进入
Obsidian工作台 → Obsidian 模板、插件工作流和工程说明
系统规则 → 最高优先级，定义边界和约束
世界观 → 世界设定，所有内容的基础
人物设定 → 角色档案，行为逻辑来源
剧情大纲 → 路线图，剧情推进依据
正文源码 → 创作层，唯一输出源
文风控制 → 风格约束，保证一致性
参考资料 → 输入层，灵感来源和非正史资料
长期记忆 → 状态层，上下文保持
技能规则 → 能力层，AI 专项技能
发布版本 → 输出层，对外发布版
素材资源 → 素材层，视觉、地图、提示词和版权来源
tools → 工具层，只读检查、统计和导出脚本
```

**设计意图**：
- AI 读取时按固定顺序，先规则再设定再正文
- 作者管理时清晰分层，不会混淆
- 修改时有明确的归属目录

---

### 2. 规则约束体系

通过规则文件约束 AI 和作者的行为边界：

| 规则文件 | 作用 |
|----------|------|
| `AI协作说明.md` | AI 读取顺序、输出要求、同步机制 |
| `最终设定硬规则.md` | 小说定位、人设、修炼体系和硬边界 |
| `当前硬边界.md` | 当前阶段禁止事项和待确认边界 |
| `正史优先级与冲突处理.md` | 设定冲突时的优先级判定 |
| `章节流程与同步规则.md` | 未来写章后的长期记忆同步要求 |

**核心理念**：创作不是自由发挥，而是在规则框架内可控创作。

---

### 3. 长期记忆机制

`长期记忆/` 维护当前状态，解决长篇创作的上下文丢失问题：

| 文件 | 作用 |
|------|------|
| `当前剧情状态.md` | 当前处于哪个剧情节点 |
| `当前人物关系.md` | 人物关系现状 |
| `当前时间线.md` | 故事内时间进度 |
| `当前世界变化.md` | 世界状态的演变 |
| `当前伏笔状态.md` | 已投放和待回收的伏笔 |
| `最近章节总结.md` | 近期剧情回顾 |
| `字数统计.md` | 工程数据 |

**设计意图**：
- AI 每次协作前读取，获得完整上下文
- 作者随时查看当前状态
- 新章节承接时不会遗漏之前的设定

## AI 协作哲学

### AI 不是替代作者，而是维护工程

AI 的职责：
- 世界观一致性检查
- 章节大纲拆解
- 伏笔投放与回收提醒
- 人物动机校验
- 正文草稿生成
- 发布版整理

### AI 能力边界原则

AI 在故事中也有边界：
- 数据不足 → 推演受限
- 算力不足 → 分析延迟
- 能源不足 → 功能降级
- 权限不足 → 信息缺失
- 未来信息残缺 → 预测不准

**设计意图**：AI 不是万能金手指，而是有边界的工具，保证故事的合理性。

## 单一创作源原则

```
正文源码/ → 唯一创作源
          ↓
      tools/export.py（自动导出）
          ↓
发布版本/ → 对外发布版
```

**设计意图**：
- 作者只维护源码
- 发布版自动生成，避免手动复制错误
- 源码可以保留批注、frontmatter、双链
- 发布版自动清理工程标记

---

## Obsidian 知识图谱

通过 `[[双链]]` 建立内容关联：

```
章节 → 人物设定 → 世界观 → 元素熵变体系
章节 → 伏笔系统 → 剧情大纲 → 主线剧情
章节 → 时间线 → 当前时间线
章节 → 世界观 → 元素熵变体系 → 逆熵
```

**作用**：
- 可视化内容关联
- 快速跳转查阅
- 保证设定一致性
- 发现遗漏和冲突

## 工程化创作流程

```
1. 读取规则 → 系统规则/
2. 读取状态 → 长期记忆/
3. 读取设定 → 世界观/、人物设定/、剧情大纲/
4. 读取文风 → 文风控制/
5. 生成草稿 → 正文源码/
6. 作者审稿 → HTML 注释批注
7. 定稿导出 → tools/export.py
8. 更新状态 → 长期记忆/
```

**设计意图**：标准化流程，每次创作都有完整上下文，不会遗漏。

## 可扩展性设计

### 新增卷

在 `tools/export.py` 中添加新卷配置：

```python
VOLUMES = [
    VolumeConfig(
        label="第三卷",
        name="新阶段",
        source_dir=ROOT / "正文源码" / "新阶段",
        ...
    ),
]
```

### 新增技能

在 `技能规则/` 添加新 skill 文件：

```markdown
# xxx.skill

## Purpose
...

## Rules
...
```

### 新增设定

按归属存放：
- 世界规则 → `世界观/`
- 人物设定 → `人物设定/`
- 剧情规划 → `剧情大纲/`
- 文风控制 → `文风控制/`

## 设计哲学总结

| 原则 | 说明 |
|------|------|
| **工程化** | 小说创作从灵感驱动转变为工程管理 |
| **规则约束** | 创作在框架内进行，避免随意偏离 |
| **分层清晰** | 内容按优先级分层，AI/作者都有固定读取顺序 |
| **长期记忆** | 状态文件保持上下文，解决长篇创作的遗忘问题 |
| **单一创作源** | 源码唯一，发布自动，避免手动错误 |
| **知识图谱** | Obsidian 双链建立关联，保证一致性 |
| **自动化** | Python 脚本处理导出、统计、清理 |
| **可扩展** | 新卷、新技能、新设定都有固定入口 |
