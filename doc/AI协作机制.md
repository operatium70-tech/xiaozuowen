# AI 协作机制

> AI 如何参与创作、读取顺序、输出要求和同步机制。

---

## AI 协作目标

AI 不是替代作者，而是帮助维护一个长期稳定的小说工程。

AI 协助完成：
- 世界观一致性检查
- 章节大纲拆解
- 伏笔投放与回收
- 人物动机校验
- 情绪节奏控制
- 正文草稿生成
- 平台发布版整理

---

## 读取顺序

写作前必须按优先级读取：

```
优先级 1: 0_系统规则/       → 规则边界
优先级 2: 7_长期记忆/       → 当前状态
优先级 3: 1_世界观/         → 世界设定
优先级 4: 2_人物设定/       → 角色档案
优先级 5: 3_剧情大纲/       → 剧情路线
优先级 6: 5_文风控制/       → 风格约束
优先级 7: 8_技能规则/       → 专项技能
```

**原则**：先规则，再状态，再设定，最后文风和技能。

---

## 输出要求

### 正文输出风格

- 冷峻
- 克制
- 压迫感
- 信息差
- 生存感
- 线悬疑

### 设定输出格式

设定输出时必须标明：

| 标记 | 说明 |
|------|------|
| 已确认 | 可直接进入正文 |
| 待确认 | 需作者确认后使用 |
| 禁止直接写入正文 | 只能作为背景或伏笔 |
| 可作为伏笔 | 可投放，但正文不直接说明 |

---

## 禁止行为

| 类别 | 禁止事项 |
|------|----------|
| 世界规则 | 不允许突然新增世界规则 |
| 人物行为 | 不允许角色严重 OOC 或强行降智 |
| 剧情推进 | 不允许靠巧合连续解决危机 |
| AI 能力 | 不允许 AI 直接替主角完成所有决策 |
| 设定 | 不允许临时添加万能能力 |

---

## AI 能力边界

AI 推演必须受限制：

| 限制 | 影响 |
|------|------|
| 数据不足 | 推演结果不完整 |
| 算力不足 | 分析延迟或简化 |
| 能源不足 | 功能降级 |
| 权限不足 | 无法获取关键信息 |
| 未来信息残缺 | 预测不准确 |
| 灾变变量不可观测 | 无法预测灾变发展 |
| 高维污染干扰 | 信息可能被污染 |

**原则**：AI 强大但有边界，不能无中生有。

## 同步机制

### 规则同步

当作者新增、修改或确认规则时，AI 必先判断影响范围：

| 规则类型 | 同步范围 |
|----------|----------|
| 工程入口规则 | `README.md` |
| AI 执行规则 | `0_系统规则/AI协作说明.md` |
| 写作与世界规则 | `0_系统规则/`、`1_世界观/` |
| 正文格式规范 | `4_正文源码/正文工程规范.md` |
| 章节规格与文风 | `5_文风控制/` |
| 剧情规划与伏笔 | `3_剧情大纲/` |
| 当前存档状态 | `7_长期记忆/` |
| AI 专项技能 | `8_技能规则/` |
| 发布与导出 | `9_发布版本/发布说明.md`、`tools/` |

### 状态同步

章节新增、定稿、删除或大改后，必须同步更新：

- 对应卷的 `卷说明.md`
- `7_长期记忆/当前剧情状态.md`
- `7_长期记忆/最近20章总结.md`
- `7_长期记忆/当前时间线.md`
- `7_长期记忆/字数统计.md`
- `3_剧情大纲/` 中相关卷纲、伏笔
- `9_发布版本/` 中对应发布文件

## 章节推进要求

每章至少推进以下一项：

| 推进项 | 说明 |
|--------|------|
| 主线危机 | 核心危机的发展 |
| 生存资源 | 主角资源变化 |
| 人物关系 | 人物关系演变 |
| 世界认知 | 对世界的理解加深 |
| AI 秘密 | AI 来源或能力揭示 |
| 灾变污染 | 灾变状态变化 |
| 伏笔投放或回收 | 伏笔状态变化 |

---

## 技能文件机制

`8_技能规则/` 存放专项写作技能：

| 技能文件 | 作用 |
|----------|------|
| `writer.skill.md` | 正文写作指南 |
| `dialogue.skill.md` | 对话风格控制 |
| `battle.skill.md` | 战斗场景写作 |
| `suspense.skill.md` | 悬疑氛围构建 |
| `emotion.skill.md` | 情绪节奏控制 |
| `pacing.skill.md` | 章节节奏控制 |
| `foreshadow.skill.md` | 伏笔投放与回收 |
| `character.skill.md` | 人物刻画规则 |

**使用方式**：AI 在执行对应任务时，读取对应技能文件。

---

## 长期记忆维护

每次完成章节、卷纲或重大设定后，必须更新 `7_长期记忆/`。

维护内容：

| 文件 | 维护内容 |
|------|----------|
| `当前剧情状态.md` | 当前剧情节点、下一步 |
| `当前人物关系.md` | 人物关系变化 |
| `当前时间线.md` | 故事内时间进度 |
| `当前世界变化.md` | 世界状态演变 |
| `当前伏笔状态.md` | 伏笔投放/回收状态 |
| `最近20章总结.md` | 近期剧情回顾 |
| `字数统计.md` | 执行 `python3 tools/word_count.py` |

## 完整协作流程

```
开始协作
    ↓
读取 0_系统规则/
    ↓
读取 7_长期记忆/
    ↓
读取 1_世界观/、2_人物设定/、3_剧情大纲/
    ↓
读取 5_文风控制/、8_技能规则/
    ↓
生成草稿
    ↓
作者审稿（批注）
    ↓
处理批注 → 更新正文
    ↓
定稿 → 执行 tools/export.py
    ↓
更新 7_长期记忆/
    ↓
更新 3_剧情大纲/（如有剧情推进）
    ↓
完成
```
