# 技术栈与工具

> 本项目使用的核心技术、工具链和自动化脚本说明。

---

## 核心技术栈

| 技术 | 用途 |
|------|------|
| **Markdown** | 所有内容的唯一格式，包括正文、设定、规则、文档 |
| **Obsidian** | 编辑器 + 知识图谱管理，通过双链建立内容关联 |
| **Python 3** | 自动化脚本：导出、字数统计、清理批注、构建全卷 |
| **Git** | 版本控制，管理创作历史和协作记录 |

---

## 目录结构设计

```
xiaozuowen/
├── 0_系统规则/      → AI 协作边界、写作规则、禁止事项、世界规则优先级
├── 1_世界观/        → 世界背景、灾变体系、AI 来源、势力、时间线
├── 2_人物设定/      → 主角、反派、配角、阵营角色与关系表
├── 3_剧情大纲/      → 总纲、卷纲、主线、支线、伏笔与结局草案
├── 4_正文源码/      → 正文章节源文件，按卷管理（唯一创作源）
├── 5_文风控制/      → 文风、对话、战斗、悬疑、节奏与漂移限制
├── 6_参考资料/      → 作品参考、氛围参考、灵感碎片
├── 7_长期记忆/      → 当前剧情状态、人物关系、世界变化、伏笔状态、字数统计
├── 8_技能规则/      → 面向 AI 的专项写作 skill 文件
├── 9_发布版本/      → 导出 txt、整卷版本与平台发布版
├── tools/           → Python 自动化脚本
├── assets/          → 地图、人物图、封面、氛围图与宣传素材
└── README.md        → 项目入口文档
```

**设计意图**：数字前缀强制排序，确保 AI 按固定优先级读取内容。

## 自动化脚本（tools/）

| 脚本 | 功能 | 使用场景 |
|------|------|----------|
| `export.py` | 导出发布版本 | 章节定稿后执行，同步到 `9_发布版本/` |
| `clean_markdown.py` | 清理 Markdown 批注 | 导出时剥离 HTML 注释、frontmatter、双链 |
| `word_count.py` | 字数统计 | 章节修改后执行，写入 `7_长期记忆/字数统计.md` |
| `build_volume.py` | 构建全卷整合版 | 导出时合并整卷章节 |
| `publish.py` | 发布流程 | 自动化发布到平台 |
| `rename_chapters.py` | 章节重命名 | 批量调整章节编号 |

## Obsidian 双链系统

正文使用 `[[双链]]` 建立内容关联：

```markdown
## 关联索引

- 主角：[[李阳]]
- 配角：[[队伍同行/林小满|林小满]]
- AI：[[AI来源]]
- 灾变：[[灾变设定]]
- 伏笔：[[伏笔系统]]
```

**作用**：
- 在 Obsidian 中生成知识图谱
- 快速跳转到相关设定
- 保持内容一致性

## YAML Frontmatter

每章正文头部使用 frontmatter 标识：

```yaml
---
类型: 正文章节
卷: 第一卷_灾变降临
章节: 第001章
标题: 十二小时预警
状态: 草稿
tags:
  - 正文
  - 第一卷
---
```

**作用**：
- Obsidian 按属性筛选章节
- 跟踪章节状态
- 统计和管理

## 发布流程

```bash
# 1. 写章节 → 4_正文源码/第一卷_灾变降临/第001章_十二小时预警.md

# 2. 定稿后导出
python3 tools/export.py

# 3. 导出结果
# → 9_发布版本/第一卷/第001章.txt
# → 9_发布版本/平台发布版/第一卷_灾变降临/第一卷_第001章_十二小时预警.txt
# → 9_发布版本/全卷整合版/第一卷_灾变降临_当前整合.txt

# 4. 同步字数统计
python3 tools/word_count.py
# → 7_长期记忆/字数统计.md
```

## 批注系统

作者审稿使用 HTML 注释：

```markdown
<!-- 批注: 这里节奏偏快，建议补一笔心理反应 -->

<!-- 批注开始: 这一段逻辑不顺 -->
原正文段落
<!-- 批注结束 -->
```

**特点**：
- 不影响 Markdown 渲染
- 导出时自动剥离
- 处理后删除

## 技术选型原则

| 原则 | 说明 |
|------|------|
| **纯文本优先** | 所有内容用 Markdown，方便 AI 读取和版本控制 |
| **单一创作源** | `4_正文源码/` 是唯一创作源，发布版由脚本自动生成 |
| **结构化分层** | 规则、设定、正文、记忆分层存放，避免混乱 |
| **自动化导出** | 减少手动操作，确保发布版一致 |
| **可追溯** | Obsidian 双链 + Git 记录，保证内容一致性 |
